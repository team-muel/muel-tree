## 1. 경험 원칙 (Feign에서 가져온 것 / 가져오지 않은 것)

**가져온 구조** — Feign이 잘하는 것은 그림체가 아니라 *사회적 무대화*다:

1. **캐릭터가 보이는 테이블** — 전원이 항상 시각 단위(토큰)로 존재. 죽음도 빈자리가 아니라 "쓰러진 토큰"으로 남는다. → `PlayerToken`.
2. **페이즈 전환 = 공간 전환** — 낮과 밤이 같은 화면의 색만 바뀌는 게 아니라 무대가 뒤집힌다. → 이중 무드 `PHASE_TONES`.
3. **정체 공개의 드라마** — 직업 공개는 정보 전달이 아니라 의식(ritual). → `RoleEmblem` + 광휘 + rise 모션.
4. **선택의 촉감** — 투표/지정은 누른 것이 빛나야 한다. → `GLOW.select*` (의미 순간에만).

**가져오지 않은 것**: 밝은 파스텔 캐릭터 그림체, 코믹 톤 전반. BoW는 어둠이 있는 세계관 — 곰돌이(장난 레이어)는 카피·이벤트에서, 화면 재질은 진지하게.

## 2. 이중 무드 — 낮↔밤

| 페이즈 | 무드 | 공간 | 강조 |
|---|---|---|---|
| lobby | dark | 따뜻한 심야 (모닥불 전) | white/40 |
| role_assign | dark | 벨벳 어둠 + 진영 광휘 | amber |
| night / night_suspect | dark | 가장 깊은 심야 + 별 | indigo |
| **day** | **light** | **따뜻한 양피지 아침** | amber-700 |
| **vote** | **light** | **황혼 — 금빛이 짙어짐** | amber-800 |
| verdict | dark | 황혼이 핏빛으로 식음 | rose |
| ended | dark | 어둠의 무대 + 진영 광휘 | 진영색 |

서사 의도: **하루의 색 곡선** = 심야(밤) → 새벽 공표(아침, 밝음) → 황혼(투표) → 핏빛(판결) → 다시 밤. 플레이어는 색만으로 "지금 어디"를 안다. 죽음 공표는 밝은 무대 위의 핏빛 카드 — 어둠 속 빨강보다 대비가 크다.

구현: `PHASE_TONES[phase] = { bg, accent, mood }`. 컴포넌트는 `MOOD[mood]` 잉크 세트(heading/body/faint/panel/chip)만 소비 — light 무드에서 흰 글자 쓰는 사고를 토큰이 차단.

## 3. 텍스처 — 광휘 (빛이 어둠 위에 선다)

정본 근거: BoW에서 **광휘 = 영성 = 마력** (`[[광휘와-대천사]]` candidate). 재질이 곧 세계관.

- **유리(Glass)**: 패널 = 반투명 + `backdrop-blur` + 상단 하이라이트 헤어라인. 떠 있는 박막.
- **광휘(Glow)**: `GLOW` 토큰 — 선택(`selectDusk`/`selectNight`), 처형(`ember`), 공개(진영 `glow`). **규율: 상시 요소에 광휘 금지.** 빛은 의미가 있을 때만 — 전부 빛나면 아무것도 빛나지 않는다.
- **결(Grain)**: 도입 보류 (후속 라운드에서 노이즈 오버레이 검토).

## 4. 시각적 매핑 — 베타 로스터 심볼 (`gomdori-role-visuals.ts`)

직업 = 기하 심볼 + 진영 색 + 모티프 한 줄. 일러스트가 생기면 `illustration` 경로만 채움 — `RoleEmblem`이 자동 교체 (호출부 무변경).

| 직업 | 심볼 | 근거 (vault 정본) |
|---|---|---|
| 대악마 | 뿔 + 낙인 원 | 코셰이+루시퍼, 메피스토 낙인 |
| 팬텀 | 일식 고리 | 침묵의 밤·일식 |
| 말렌 | 나란한 두 혼불 | 악령 마야 |
| 베스토 | 갈라진 가면 | 두 번째 자아 (솔/하베스토) |
| 가인 | 가려진 눈 | 진실을 가리는 암흑 |
| 루나 | 초승달 + 우물 방울 | 달빛이 비치는 우물 (아서의 거울 짝) |
| 로건 | 금 간 펜던트 | 부서진 펜던트 |
| 엘런 | 기울어진 저울 | 박해자 — 불공정 |
| 로마즈 | 육각 배지 | 무죄추정의 원칙 (경찰) |
| 라이너 | 세 줄 발톱 | 수호신 백호 |
| 도르단 | 돋보기 | 침착한 탐정 |
| 하브레터스 | 빛 갈래 십자 | 의사 선서 (사용자 확정 모티프) |
| 미즐렛 | 디저트 잔 + 반짝 | 행복을 파는 가게 |
| 헬렌 | 별 + 잠든 호 | 황금빛 수면 |
| 우노 | 깃발 | 군인의 사명·명예 |
| 아서 | 검 + 잔불 점 | 잔불 대검 (해 ↔ 루나) |
| 세이카 | 8방 초신성 | 초신성 — 세야카와 별빛 (자매 짝) |
| 루루 | 8분음표 | 소나타·매료 |
| 파스아 | 후광 원 | 사이비 교주 — 구원자 |
| 전향자 | 나선 | 안으로 감기는 길 |

짝 관계의 시각 운율 (후속): 아서(검+잔불, amber) ↔ 루나(달+우물, indigo) / 세이카(초신성) ↔ 세야카(미구현, 추락하는 별 예약) — 매핑 짝은 심볼 구조를 거울로.

## 5. 컴포넌트 계약

- `RoleSymbol` — 24×24 stroke 1.6 round. 심볼만, 데이터 없음.
- `RoleEmblem(role, size, mood, glow)` — 심볼+색+광휘 링. `illustration` 슬롯 처리.
- `PlayerToken(name, avatarUrl, alive, selected, selectedGlow, mood)` — 테이블의 기본 단위. 아바타 우선, 없으면 이니셜. 사망 = ✕ + 탈색 (자리 유지).
- 페이즈 컴포넌트는 색을 직접 쓰지 않는다 — `MOOD`/`GLOW`/`FACTION_COLORS`/`PHASE_TONES` 토큰만. (하드코딩 금지 — 결정 잠금 #4와 같은 원칙.)
