## 1. 경험 원칙 (Feign에서 가져온 것 / 가져오지 않은 것)

**가져온 구조** — Feign이 잘하는 것은 그림체가 아니라 *사회적 무대화*다:

1. **캐릭터가 보이는 테이블** — 전원이 항상 시각 단위(토큰)로 존재. 죽음도 빈자리가 아니라 "쓰러진 토큰"으로 남는다. → `PlayerToken`.
2. **페이즈 전환 = 공간 전환** — 낮과 밤이 같은 화면의 색만 바뀌는 게 아니라 무대가 뒤집힌다. → 이중 무드 `PHASE_TONES`.
3. **정체 공개의 드라마** — 직업 공개는 정보 전달이 아니라 의식(ritual). → `RoleEmblem` + 광휘 + rise 모션.
4. **선택의 촉감** — 투표/지정은 누른 것이 빛나야 한다. → `GLOW.select*` (의미 순간에만).

**가져오지 않은 것**: 밝은 파스텔 캐릭터 그림체, 코믹 톤 전반. BoW는 어둠이 있는 세계관 — 곰돌이(장난 레이어)는 카피·이벤트에서, 화면 재질은 진지하게.

## 2. 이중 무드 — 낮↔밤

| 페이즈 | 무드 | 공간 | 강조 |
|---|---|---|---|
| lobby | dark | 따뜻한 심야 (모닥불 전) | white/40 |
| role_assign | dark | 벨벳 어둠 + 진영 광휘 | amber |
| night / night_suspect | dark | 가장 깊은 심야 + 별 | indigo |
| **day** | **light** | **따뜻한 양피지 아침** | amber-700 |
| **vote** | **light** | **황혼 — 금빛이 짙어짐** | amber-800 |
| verdict | dark | 황혼이 핏빛으로 식음 | rose |
| ended | dark | 어둠의 무대 + 진영 광휘 | 진영색 |

서사 의도: **하루의 색 곡선** = 심야(밤) → 새벽 공표(아침, 밝음) → 황혼(투표) → 핏빛(판결) → 다시 밤. 플레이어는 색만으로 "지금 어디"를 안다. 죽음 공표는 밝은 무대 위의 핏빛 카드 — 어둠 속 빨강보다 대비가 크다.

구현: `PHASE_TONES[phase] = { bg, accent, mood }`. 컴포넌트는 `MOOD[mood]` 잉크 세트(heading/body/faint/panel/chip)만 소비 — light 무드에서 흰 글자 쓰는 사고를 토큰이 차단.

## 3. 텍스처 — 광휘 (빛이 어둠 위에 선다)

정본 근거: BoW에서 **광휘 = 영성 = 마력** (`[[광휘와-대천사]]` candidate). 재질이 곧 세계관.

- **유리(Glass)**: 패널 = 반투명 + `backdrop-blur` + 상단 하이라이트 헤어라인. 떠 있는 박막.
- **광휘(Glow)**: `GLOW` 토큰 — 선택(`selectDusk`/`selectNight`), 처형(`ember`), 공개(진영 `glow`). **규율: 상시 요소에 광휘 금지.** 빛은 의미가 있을 때만 — 전부 빛나면 아무것도 빛나지 않는다.
- **결(Grain)**: 도입 보류 (후속 라운드에서 노이즈 오버레이 검토).

## 4. 시각적 매핑 — 베타 로스터 심볼 (`gomdori-role-visuals.ts`)

직업 = 기하 심볼 + 진영 색 + 모티프 한 줄. 일러스트가 생기면 `illustration` 경로만 채움 — `RoleEmblem`이 자동 교체 (호출부 무변경).

| 직업 | 심볼 | 근거 (vault 정본) |
|---|---|---|
| 대악마 | 뿔 + 낙인 원 | 코셰이+루시퍼, 메피스토 낙인 |
| 팬텀 | 일식 고리 | 침묵의 밤·일식 |

| 가인 | 가려진 눈 | 진실을 가리는 암흑 |
| 루나 | 초승달 + 우물 방울 | 달빛이 비치는 우물 (아서의 거울 짝) |
| 로건 | 금 간 펜던트 | 부서진 펜던트 |
| 엘런 | 기울어진 저울 | 박해자 — 불공정 |
| 로마즈 | 육각 배지 | 무죄추정의 원칙 (경찰) |
| 라이너 | 세 줄 발톱 | 수호신 백호 |
| 도르단 | 돋보기 | 침착한 탐정 |

## 5. 컴포넌트 계약

- `RoleSymbol` — 24×24 stroke 1.6 round. 심볼만, 데이터 없음.
- `RoleEmblem(role, size, mood, glow)` — 심볼+색+광휘 링. `illustration` 슬롯 처리.
- `PlayerToken(name, avatarUrl, alive, selected, selectedGlow, mood)` — 테이블의 기본 단위. 아바타 우선, 없으면 이니셜. 사망 = ✕ + 탈색 (자리 유지).
- 페이즈 컴포넌트는 색을 직접 쓰지 않는다 — `MOOD`/`GLOW`/`FACTION_COLORS`/`PHASE_TONES` 토큰만. (하드코딩 금지 — 결정 잠금 #4와 같은 원칙.)

## 7. 2차 — 무대 아키텍처 (Feign 강화, 2026-06-11)

사용자 피드백: Feign을 더 강하게. 모바일 지원 구조. 친구 부르기 = 모바일 하단 시트. 상태("자유 토론 시간")는 하단에 직관 표시. 투표는 페이지 전환 없이 창.

**구조 전환: 페이지 교체 → 무대 위 레이어.**

```
GameStage   — 상시 테이블 (전원 토큰, 죽음 = 쓰러진 토큰. 무대 바닥 광)
StatusDock  — 하단 고정 독: 페이즈 라벨 + 상태 한 줄 + 일차 + 타이머 + 내 직업
BottomSheet — 보조 패널: 데스크톱 = 사이드, 모바일 = 하단에서 올라오는 시트
ActionModal — 무대 위 창: 투표/의심 확정. 배경 옅게만 — 무대는 계속 보인다
```

페이즈 적용: 로비 = 무대(참가자) + 친구부르기 시트 / 아침 = 공표 배너 + 무대, 상태는 독("자유 토론 시간") / 투표·의심 = **무대에서 직접 지목 + 창에서 확정** (선택 광휘: 황혼/인디고) / 사망자 = 관전 피드 시트.

레이어 규약: 무대(z0) < 독(z30) < 시트/창(z40) < 페이즈 전환 오버레이(z50, GameFrame announce). 모바일: 창은 하단 정렬(독 위 pb-20), 시트 peek은 bottom-16 (독과 자리 공유 — 동시 출현은 로비·사망 관전뿐).

비범위(후속): NightPhase 무대화 (능력 지정도 무대 지목으로 — 별도 PR) / 무대 토큰 모션 (입장·사망 쓰러짐) / 시트 드래그 제스처 (현재 탭 토글).


### 7.1 완성분 (2026-06-11, "ALL" 지시)

- **NightPhase 무대화**: 밤 능력도 무대 지목 + 창 확정. 다중 능력(팬텀 악몽·봉인·일식 / 말렌 방출·빙의 / 베스토 히든·변신)은 창 안의 **능력 칩**으로 전환, 각 능력 독립 제출(기존 SecondaryAbility 의미 보존). 악마 채팅 = BottomSheet. 조사 결과·복원·의심 봉인·첫 밤·취침 전부 무대 위 창으로 통일. 대상 조건은 `GameStage.canSelect` 주입(부활=탈락자만, 처치=악마팀 제외, 치료=자신 포함).
- **토큰 모션**: 입장 fade+zoom / 죽음 = 쓰러짐(rotate-12 + 가라앉음 + grayscale, transition 500ms — alive 플래그 반전 시 자동 재생) / ✕ 가 늦게 떠오름(700ms). reduced-motion 시 정적.

## 8. 3차 — 디스플레이 적응 · 배회 무대 · 진입/로비 개편 (2026-06-11)

사용자 피드백: ① Activity가 디스플레이 크기에 맞춰 동적 렌더링 ② 모바일은 *별도 구조* ③ 초기 로비(랜딩)를 인게임 수준으로 ④ preview에서 "아바타가 돌아다니는 Feign 최소구조"가 식별 안 됨.

**디스플레이 적응 (`src/lib/game/display.tsx`)**
- `DisplayProvider` 가 루트 폰트 크기를 뷰포트에 비례 조정 — 게임 UI는 rem 기반(Tailwind)이라 무대 전체가 균일 스케일. 데스크톱 기준 1280×760=1.0 (0.8~1.5 클램프), 모바일 기준 폭 400 (0.9~1.2). 게임 라우트에서만 마운트·언마운트 시 복원.
- `useDisplay().layout` = `mobile | desktop`. Discord 런치 파라미터 `platform=mobile` 1순위, 아니면 폭<768 (PiP 포함 — 구조는 공간을 따른다). BottomSheet·LobbyPhase·LandingScreen 이 CSS 브레이크포인트가 아니라 이 신호로 *구조 분기*.
- px 고정값은 의도적으로 비스케일(헤어라인·블러). 페이즈 내 텍스트/사이즈 px 잔재는 rem 으로 이행 완료.
- GameFrame: `h-full + overflow-y-auto + m-auto` — ActivityLayout 의 `overflow-hidden` 클리핑(작은 디스플레이에서 내용 잘림) 해소.

**배회 무대 (Feign 최소구조의 가시화)**
- `GameStage.roam` — 생존 토큰이 제자리 주변을 느리게 배회 (`gomdori-roam-a/b/c`, ±12px, 18~24s, 음수 delay 위상차). 지목 무대에서는 금지 (조준 안정성).
- `PlayerToken.chrome=false` — 카드 없이 아바타(+1단계 큼)+이름만: 무대에 "서 있는" 캐릭터. 로비·랜딩 전용, 지목 그리드는 카드 유지.
- 죽음은 roam 제외 — 쓰러진 토큰은 그 자리에 멈춰 잔류.

**진입(랜딩)·로비**
- `LandingScreen`(분리·개편): 키 아트 + *인스턴스 참가자가 입장 전부터 Discord 아바타로 무대 배회* + 만들기/참가. `InstanceParticipant.avatar` 해시 → `participantAvatarUrl()` (CDN — 매치 avatar_url 과 같은 CSP 경로).
- `LobbyPhase`: 패널 중심 → 인게임 문법 (공표 배너형 헤더 + 칩, 열린 무대 roam, 중앙 액션). 캡션 명단 제거 — 상태가 캐릭터 발밑 sub 로.

**preview 작업대**
- 갱신 식별: 헤더·푸터에 빌드 스탬프 (`NEXT_PUBLIC_VERCEL_GIT_COMMIT_SHA`).
- 섹션 00 "무대 최소구조 (Feign)" — 배회 무대 단독. 섹션 01 "랜딩". 목 플레이어 8인 전원 *아바타 형태*(data-URI 실루엣 — `<img>` 렌더 경로 검증) 보유.
